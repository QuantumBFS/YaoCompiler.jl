YaoCompiler.NativeJuliaTarget — Type
NativeJuliaTarget <: YaoCompileTarget

This target assumes one compiles to native Julia, thus the measurement type will be directly inferred from native intrinsics.

source
YaoCompiler.routine_stub — Function
routine_stub(routine, args...)

A function to store the raw CodeInfo of defined routines to support multiple dispatch. We don't directly define main gate, ctrl etc. because we can't support multiple dispatch on them directly.

source

YaoCompiler.@gate — Macro
@gate <locs> => <gate>

Syntax sugar for apply(gate, locs), must be used inside @device. See also @device.

Tips

You don't have to write @gate in most cases inside @device. But in case there is ambuigity, you can annotate the expression with @gate explicitly.
